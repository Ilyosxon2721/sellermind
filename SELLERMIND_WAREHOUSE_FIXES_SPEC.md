## 2. Задача 1: Пагинация в разделе "Остатки"

### 2.1 Описание

Добавить пагинацию на страницу "Остатки на складе". Сейчас все записи выводятся списком без разбивки на страницы.

### 2.2 Требования

- Показывать по 50 записей на странице (настраиваемо: 25/50/100)
- Пагинация внизу таблицы
- Сохранять фильтры при переходе между страницами
- Показывать общее количество записей

### 2.3 UI пагинации

```
Показано 1-50 из 145 записей    [25 ▼]    [< Назад] [1] [2] [3] [Вперёд >]
```

### 2.4 Реализация (Livewire)

```php
// В компоненте Livewire
use WithPagination;

public int $perPage = 50;

public function render()
{
    $stocks = Sku::query()
        ->where('company_id', auth()->user()->company_id)
        ->where('warehouse_id', $this->warehouseId)
        ->when($this->search, function ($q) {
            $q->where('sku_code', 'like', "%{$this->search}%")
              ->orWhere('barcode_ean13', 'like', "%{$this->search}%");
        })
        ->when($this->categoryId, function ($q) {
            $q->whereHas('product', fn($q) => $q->where('category_id', $this->categoryId));
        })
        ->with(['product', 'productVariant', 'warehouse'])
        ->paginate($this->perPage);

    return view('livewire.stock.index', compact('stocks'));
}
```

### 2.5 Blade

```blade
{{-- Таблица --}}
<table>
    @foreach($stocks as $stock)
        <tr>...</tr>
    @endforeach
</table>

{{-- Пагинация --}}
```

## 4. Задача 3: Автозагрузка остатков

### 4.1 Описание

Убрать кнопку "Загрузить" — остатки должны загружаться автоматически при:
- Открытии страницы
- Изменении склада
- Изменении фильтров

### 4.2 Текущее поведение (неправильно)

```
1. Пользователь открывает страницу
2. Таблица пустая
3. Пользователь нажимает "Загрузить"
4. Данные появляются
```

### 4.3 Требуемое поведение (правильно)

```
1. Пользователь открывает страницу
2. Данные загружаются автоматически
3. При смене фильтра — данные обновляются автоматически
```

### 4.4 Реализация

```php
// Livewire компонент
public function mount()
{
    // Установить склад по умолчанию
    $this->warehouseId = Warehouse::where('company_id', auth()->user()->company_id)
        ->first()?->id;
    
    // Данные загрузятся автоматически в render()
}

// Убрать метод loadData() если он есть
// Убрать кнопку "Загрузить" из Blade

public function render()
{
    // Данные всегда загружаются
    $stocks = Sku::query()
        ->where('company_id', auth()->user()->company_id)
        ->when($this->warehouseId, fn($q) => $q->where('warehouse_id', $this->warehouseId))
        ->when($this->search, fn($q) => $q->where('sku_code', 'like', "%{$this->search}%"))
        ->paginate($this->perPage);
        
    return view('livewire.stock.index', compact('stocks'));
}
```

## 8. Задача 7: Исправление данных в "Журнале движений"

### 8.1 Описание

Страница "Журнал движений" показывает неправильные данные. Нужно изучить модуль и исправить.

### 8.2 Что видно на скриншоте

| Поле | Значение | Проблема? |
|------|----------|-----------|
| Дата | 07.02.2026, 11:22 | ОК |
| Документ | — (пусто) | ❓ Должен быть документ? |
| Тип | Резерв | ОК |
| № заказа | 4590848100 | ОК |
| Маркетплейс | Wildberries | ОК |
| Магазин | OOO RIS ONLINE | ОК |
| Тип заказа | Маркетплейс | ОК |
| Склад | Основной склад | ОК |
| SKU | FH2020001-L-PASTKOR | ОК |
| Δ КОЛ-ВО | -1 | ОК (резерв уменьшает) |

### 8.3 Возможные проблемы

1. **Пустой документ** — возможно не привязывается документ к движению
2. **Неправильные связи** — движение не связано с правильным заказом
3. **Дублирование** — одно движение отображается несколько раз
4. **Неправильный знак** — плюс/минус неправильно рассчитывается
