Если запрос магазина не удалось выполнить, Маркет возвращает в ответ параметр errors. Он содержит коды возникших ошибок (параметр code) и их краткие описания (параметр message).

Если ошибок с определенным кодом станет слишком много, вы получите уведомление об этом в кабинете продавца на Маркете.

Код

Название

Что случилось и что делать

400

Bad Request

Что-то не так в содержании запроса. Например, ошибка в формате JSON-данных или вы пытаетесь изменить статус отмененного или доставленного заказа.

Ориентируйтесь на описание ошибки, чтобы понять, что именно не так.

401

Unauthorized

В запросе не указан (или указан, но не там) токен авторизации.

403

Forbidden
  • Токен авторизации не сработал. Для OAuth-токена: скорее всего, у него истек срок годности или вы удалили из числа сотрудников человека, на которого был оформлен токен.
  • Методы API недоступны из-за неактивности магазина или кабинета.
  • Другая ошибка доступа.

404

Not Found
  • Запрашиваемый метод или ресурс не найден. Проверьте адреса запросов, по которым обращается система магазина.
  • Указанный параметр не найден. Например, кампания. Проверьте корректность передаваемых данных.
  • Версия метода не найдена. Проверьте корректность используемой версии.

405

Method Not Allowed

На указанном ресурсе нет такого метода.

Проверьте адреса запросов, по которым обращается система магазина.

415

Unsupported Media Type

Запрашиваемый тип контента не поддерживается методом.

Проверьте корректность запроса.

420

Enhance Your Calm

Магазин превысил лимит запросов.

Убедитесь, что ваша система не отправляет ничего лишнего.

423

Locked

Метод невозможно использовать для этого магазина.

Ориентируйтесь на описание ошибки, чтобы понять, что именно не так.

499

Client Closed Request

Клиент закрыл соединение до окончания обработки запроса на стороне Маркета. Проверьте таймауты на стороне клиента и повторите запрос.

500

Internal Server Error

Внутренняя ошибка Маркета.

503

Service Unavailable

Сервер Маркета перегружен.
